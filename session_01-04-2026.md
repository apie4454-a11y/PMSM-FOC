## Lessons Learned (This Session)

1. **Simulation ripple ≠ problem**
   - High-frequency ripple from PWM is expected
   - Verify it's at switching frequency (20 kHz)
   - If stable + constant amplitude → OK

2. **MTPA validation is simple but powerful**
   - Just verify id ≈ 0 throughout operation
   - If it holds, you're extracting max torque/ampere
   - One plot (id over time) proves entire strategy

3. **Test plan documentation is resume gold**
   - 16 test scenarios + pass/fail criteria = professional rigor
   - Graph capture list (40-50 files) = proof trail
   - Overlay comparisons (SVPWM vs ST) = differentiator

4. **Anti-windup testing is non-negotiable**
   - Most engineers skip this
   - First question in interviews: "What happens during overload?"
   - Your answer: "Here's the test result showing graceful degradation"

5. **Discrete controller architecture is NON-NEGOTIABLE for digital systems**
   - Microcontroller firmware ALWAYS runs discrete (20kHz, 50μs time steps)
   - Continuous PI controllers in Simulink ≠ discrete controllers in firmware
   - Mismatch → simulation won't predict hardware behavior
   - Solution: Use Discrete Integrator (Ts=50μs, Ki_discrete=5.2779) + Gain blocks
   - To capture PWM transients: Solver = fixed-step ode4, step size = PWM_period/4
   - Universal pattern: Control updates align with PWM cycles, solver integrates between them

   **MATLAB Implementation Steps:**
   - **Step 1: Set Power GUI sample time**
     - Double-click Power GUI block
     - Find "Sample time [s]:" field → Enter `5e-5` (for 20kHz PWM)
     - Click Apply/OK
   - **Step 2: Configure Solver**
     - Go to Simulation → Model Configuration Parameters (Ctrl+E)
     - Solver tab: Type = `Fixed-step` (NOT variable-step)
     - Solver = `ode4` (Runge-Kutta 4th order)
     - Max step size = `1.25e-5` (4 subdivisions of 50μs PWM period)
     - Click OK
   - **Step 3: Set Controller Sample Times**
     - For each Discrete Integrator block: Set `Ts = 5e-5`
     - For each discrete Gain block paired with integrator: No Ts needed (inherits)
     - Verify: Double-click block → Parameters tab → Ts value shows correctly

6. **Cascading subtraction in nested loops = instability**
   - WRONG: Speed PI outputs torque → subtract Te_actual → calculate iq_error → current PI
   - Problem: Double feedback at different levels fighting each other
   - Result: Oscillations, controller fighting itself, slow response (~0.34 Nm stall limit instead of 0.067 Nm)
   - FIX: Clean cascade with single feedback per stage
   - Speed loop: outputs iq_ref only (no TeActual subtraction)
   - Current loop: uses iq_error only (not Te_error)
   - Eliminates interference between control layers ✓

7. **Continuous → Discrete PI gain conversion (one-line formula)**
   - Kp: Stays the same (no time dependence)
   - Ki: **Multiply by Ts** or **divide by sampling frequency**
   - Formula: Ki_discrete = Ki_continuous × (1/f_sampling) = Ki_continuous / f_Hz
   - Example: Ki=105,558 @ 20kHz → Ki_discrete = 105,558 / 20,000 = 5.2779
   - For speed loop @ 2kHz: Ki=1.257e-3 → Ki_discrete = 1.257e-3 / 2,000 = 6.285e-7
   - Applies to any cascade controller with different loop rates
