/* ========== FLOATING POINT PRECISION ========== */

/*
Forward Euler Integration:
  x_new = x + dx_dt * Ts
  
Issues:
  1. Accumulation error: Over 30s at 20kHz = 600k steps
  2. Small timestep (5e-7s): dx_dt * Ts can be very small
  3. Float precision: 24 bits = ~7 decimal digits
  
MOTOR INTEGRATION (5e-7s timestep):
  id_new = id + di_d_dt * 5e-7
  
  Example: di_d_dt = 100 A/s
           Δid = 100 * 5e-7 = 5e-5 A per step
           Over 20 steps per 20kHz cycle: 0.001 A
  
  ✅ ACCEPTABLE: Timestep is fine, error is << motor dynamics

FOC CONTROL (5e-5s timestep):
  PIcontroller much slower, 5e-5 >> motor electrical time constant
  
  ✅ ACCEPTABLE
*/

/* ========== EDGE CASES IN TRANSFORMS ========== */

/*
1. CLARKE TRANSFORM EDGE CASES:
   Input: va=Vdc, vb=0, vc=0 (single-phase worst case)
   
   Expected: α = (2/3)*Vdc - (1/3)*0 - (1/3)*0 = (2/3)*Vdc ✅
             β = (√3/2)*0 - (√3/2)*0 = 0 ✅
   
   Input: va=vb=vc=0 (no voltage)
   Output: α=β=0 ✅ (saturates correctly)
   
   Input: va=vb=vc=Vdc (DC offset)
   Output: α=(2/3)*Vdc - (1/3)*Vdc - (1/3)*Vdc = 0 ✅
           β=0 ✅ (cancels common mode correctly)

2. PARK TRANSFORM EDGE CASES:
   Input theta=0: [d,q] = [α,β]
   
   Code: d = cos(0)*α + sin(0)*β = α ✅
         q = -sin(0)*α + cos(0)*β = β ✅
   
   Input theta=π/2: [d,q] rotated 90°
   
   Code: d = cos(π/2)*α + sin(π/2)*β = β ✅
         q = -sin(π/2)*α + cos(π/2)*β = -α ✅
   
   Input theta=2π: Full rotation back to start
   
   Code: Wrapped in motor_model.c - ✅ CORRECT
         theta_ele = mod(theta_ele, 2π)

3. INVERSE PARK EDGE CASES:
   Input d=1, q=0, theta=0: [α,β] = [1,0]
   
   Code: α = cos(0)*1 - sin(0)*0 = 1 ✅
         β = sin(0)*1 + cos(0)*0 = 0 ✅
   
   Input d=0, q=1, theta=π/2: [α,β] = [1,0] in original frame
   
   Code: α = cos(π/2)*0 - sin(π/2)*1 = -1 ⚠️ Should be +1 after full rotation
         (Actually correct; depends on rotation direction definition)

4. INVERSE CLARKE EDGE CASES:
   The dq_to_abc function uses simplified form assuming balanced 3-phase
   
   va = (2/3)*α
   vb = -(1/√3/2)*α + (1/2)*β
   vc = -(1/√3/2)*α - (1/2)*β
   
   Check: va + vb + vc = 0 ? (balanced)
          (2/3)*α - (1/√3)*α = (2/3 - √3/3)*α
          = (2/3 - 0.577)*α = 0.089*α
   
   ❌ NOT EXACTLY ZERO - small error term persists
   
   However, in practice this is never used in your code:
   - Voltages come from Park inverse (inverse_park_transform)
   - Motor model works in dq, outputs converted internally
   - No explicit dq_to_abc used in FOC control
   
   ⚠️ WARNING: dq_to_abc is provided but not critical path
*/
